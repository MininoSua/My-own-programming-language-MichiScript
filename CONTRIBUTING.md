# 🐾 Contributing to MichiScript

Meow! Thank you for your interest in improving **MichiScript**. This is an open-source project, and all paws are welcome to help us grow.

## 🛠️ How Can You Help?

You can contribute in several ways:
1.  **Reporting Bugs:** If the Lexer, Parser, or Evaluator crashes, please open an "Issue."
2.  **Suggesting Features:** Want MichiScript to have networking support or advanced math? Let us know!
3.  **Improving Code:** Fix syntax errors (like the current nested `sino` block issue) or optimize performance.

## 📜 Golden Rules

To keep the code clean and functional, please follow these guidelines:

*   **Mandatory Verification:** Before submitting any changes, ensure the code **works** and does not break existing examples in the `examples/` folder.
*   **Clean Code:** Maintain a readable, well-commented style. Avoid "ugly" or confusing logic.
*   **Naming Conventions:** If you add native functions, try to use fun, cat-related names (e.g., `miau_hora`, `rascar`, `ronronear`).

## 🚀 Steps to Submit a Pull Request (PR)

1.  **Fork** the repository.
2.  Create a branch for your improvement: `git checkout -b feature/amazing-improvement`.
3.  Make your changes and write a clear commit message: `git commit -m 'Fixed syntax error in the Parser'`.
4.  Push your changes to your fork: `git push origin feature/amazing-improvement`.
5.  Open a **Pull Request** toward the `main` branch of this repository.

## 🐛 Priority Issues

We are currently looking for help with:
*   **Nested `sino si` Support:** The current Parser occasionally misinterprets `{}` braces in complex conditional structures.
*   **Evaluator Optimization:** Making `mientras` (while) loops execute faster.
